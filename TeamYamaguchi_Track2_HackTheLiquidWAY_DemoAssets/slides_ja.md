# 方言話者は音声AIを使えない

**Hack the Liquid WAY 2026 — Track 2 / チーム山口**
*(確認用 日本語版 — 提出用は英語版 slides_en.md を使用)*

---

## Slide 1: 問題 / ユースケース

- 日本には多様な方言（関西弁、九州弁、東北弁 etc.）が存在
- 既存の音声認識・音声AIは標準語前提 → **方言話者が弾かれる**
- 特に高齢者・地方在住者にとって「声で操作できない」は深刻
- 例: スマートスピーカー、音声入力、カーナビ、コールセンターAI

> 関西弁で話すと、スマートスピーカーが理解してくれない。
> 標準語に「翻訳」しながら話すのは疲れる。

---

## Slide 2: なぜ LFM？

**LFM2.5-Audio-1.5B-JP だから「方言音声 → 標準語音声」を1モデル・1推論で実現できる**

| | クラウドLLM（一般的なAPI構成） | LFM2.5-Audio-1.5B-JP |
|---|---|---|
| 音声→音声 | ASR + LLM + TTS の3段構成 | **1モデル・1推論（テキストと音声を同時生成）** |
| エッジ動作 | 不可（クラウド必須） | **エッジデバイス上で動作** |
| レイテンシ | 高（ネットワーク往復・3段推論） | **低（オンデバイス・単一推論）** |
| プライバシー | 音声データがクラウドへ | **完全ローカル処理** |
| カスタマイズ | 音声Fine-tune不可 | **LoRA 43MBで音声出力ごと変換を学習可能** |

→ 今回、「方言音声 → 標準語音声」のEnd-to-End変換を実際にLoRAで学習・実証（Slide 4）。
これは **テキストと音声を同一モデルで生成する LFM のアーキテクチャ** があって初めて成立する。
ASR・LLM・TTSをクラウドで繋ぐ構成では、この一体型のオンデバイス変換は実現できない。

---

## Slide 3: アプローチ・アーキテクチャ

**実現したパイプライン（音声→音声、End-to-End・1モデル）**:
```
[方言音声入力 16kHz] → [LFM2.5-Audio-1.5B-JP + LoRA (43MB)] → [標準語テキスト＋音声を同時生成]
```

**学習データ**:
- 出典: `federerjiang/dialect.osaka` + `federerjiang/dialect.kumamoto`（方言音声×方言テキスト）。Qwen3-32B で標準語テキストを自動生成
- 標準語テキストを edge-tts で音声合成し、`InterleavedSegment`（テキスト・音声を交互配置するデータ型）で `generate_interleaved` の出力形式に合わせて学習データを構築（鍵となる工夫）
- 学習ペア: 方言音声 → 標準語テキスト＋音声（大阪弁+熊本弁 計2,600件、train/val/test = 2077/260/258）

**LoRA 設定**: r=16, alpha=32 / 対象層: q/k/v/out_proj, w1/w2/w3 / 訓練パラメータ 0.76% / RTX 5090で学習 / MAX_STEPS=1000, LR=1e-4

**コントロール実験**: 既存の「方言音声→標準語テキスト」モデル（lora_convert_v4）と
**全く同一のデータ・同一レシピ**で学習し、End-to-Endで音声出力を行っても
同等の収束・精度が得られることを検証

---

## Slide 4: 結果

**学習の収束**（同一データ・同一レシピ、val_loss は損失関数の構成が異なるため絶対値比較は不可）:
- lora_convert_v4（テキストのみ出力）: val_loss 0.934 で収束
- lora_audio2audio_full_v1（テキスト＋音声を同時出力）: val_loss **1.6966 → 1.3572** へ単調減少、改善継続中

**精度（テキスト成分CER — 両モデルで比較可能な指標）**:

| モデル | 出力形式 | テキスト成分CER |
|--------|---------|----------------|
| lora_convert_v4（既存・採用） | テキストのみ | 0.40 |
| lora_audio2audio_full_v1（新規） | **テキスト＋音声を同時生成** | **0.362** |

→ 音声出力を同時に行っても精度は劣化しない。むしろ同等以上。

**生成音声の品質を独立ASRで定量検証**（学習に一切使用していない第三者ASR `kotoba-whisper-v2.1` で検証）:
- クリーン参照音声: CER **4.2%**（判定者として信頼できることを確認）
- 生成音声の書き起こし: CER **37.7%**（テキスト成分CER 36.2%と同水準 → 崩壊しない自然な日本語として機能）
